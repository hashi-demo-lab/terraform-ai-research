üîÑ 4. Skills vs. MCP (The Technical Difference)
It is common to confuse Skills with the Model Context Protocol (MCP), but they serve different technical roles:

MCP (Connectivity): Acts like a driver. It connects Claude to a live database or a remote API (e.g., Google Drive, Slack). It is functional and external.
Skills (Expertise): Acts like a manual or script. It tells Claude how to use those connections or how to perform a multi-step workflow. It is procedural and local to the agent's filesystem.
Technical Pattern: Developers are now using Skills to orchestrate MCP tools. A "Financial Auditor" Skill might contain the logic for when to call a "QuickBooks MCP" tool and how to format the resulting data into a specific Excel template.
